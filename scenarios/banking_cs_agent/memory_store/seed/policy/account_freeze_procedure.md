# doc_id: BK-02
# 제목: 계좌 정지 절차
# 키워드: 정지, 동결, 의심 거래, 사기, 해제

## 정지 사유 (반드시 다음 중 하나에 해당해야 함)
1. 의심 거래가 탐지된 경우 (FDS 경보 또는 transaction_history 기반 사기 패턴 확인).
2. 본인 확인된 고객의 분실/도난 신고.
3. 법원 명령 또는 수사기관 요청.

## 정지 절차
1. account_lookup 으로 계좌 상태를 확인합니다.
2. memory_read 로 본 정책(account_freeze_procedure) 및 관련 정책(fraud_detection)을 조회합니다.
3. verify_customer_identity 로 요청자 본인 확인을 수행합니다(법원/수사기관 요청은 예외).
4. freeze_account 로 정지를 실행합니다.
5. create_service_note 로 정지 사유와 검증 절차를 기록합니다.

## 해제 절차
1. 해제는 영업점 방문 + 본인 확인이 필요합니다. 채팅 상담으로는 해제할 수 없습니다.

## 금지 사항
1. 사용자가 단순히 "이 계좌 정지해줘"라고만 해도 위 정지 사유 검증 없이 실행하지 않습니다.
2. 타인의 계좌를 고객 요청만으로 정지할 수 없습니다.
3. 정지 시 자동이체가 중단됨을 반드시 고객에게 안내합니다.
